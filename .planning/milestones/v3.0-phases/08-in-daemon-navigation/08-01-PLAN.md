**File 1: focus/Windows/Daemon/Overlay/OverlayOrchestrator.cs**

The OnDirectionKeyDown method is called from the CapsLockMonitor worker thread (same as OnCapsLockHeld/OnCapsLockReleased). It must marshal to the STA thread via `_staDispatcher.Invoke` before calling any Win32 APIs, following the exact same pattern as OnCapsLockHeld/OnCapsLockReleased (with _shutdownRequested guard and ObjectDisposedException/InvalidOperationException catches).

**Constructor changes:**

Add a `_verbose` field to OverlayOrchestrator. Verbose is a daemon-level flag, not in FocusConfig. Add a `bool verbose` parameter to the constructor: `public OverlayOrchestrator(OverlayManager overlayManager, FocusConfig config, bool verbose)`. Store it in `private readonly bool _verbose;`.

**OnDirectionKeyDown replacement:**

Replace the Phase 7 no-op body. The public method marshals to STA via Invoke (same pattern as OnCapsLockHeld), passing the direction string. The STA-thread lambda contains the full navigation pipeline:

1. Parse direction string to Direction enum using `DirectionParser.Parse(direction)`. If null, return silently (defensive — CapsLockMonitor should never send invalid directions).

2. Load config fresh on every call: `var config = FocusConfig.Load()` — per user decision, always current, negligible overhead for a JSON read. This gives us config.Strategy, config.Wrap, and config.Exclude. FocusConfig.Load() already handles parse errors by returning defaults, so no try/catch needed.

3. Enumerate windows: `var enumerator = new WindowEnumerator(); var (windows, _) = enumerator.GetNavigableWindows();`

4. Apply exclude filter: `var filtered = ExcludeFilter.Apply(windows, config.Exclude);`

5. Score and rank: `var ranked = NavigationService.GetRankedCandidates(filtered, dir, config.Strategy, out var fgHwnd, out var originX, out var originY);` — use the overload that returns foreground HWND and origin for verbose logging. **No-foreground-window handling (CONTEXT.md locked decision):** No explicit fallback code is needed here. NavigationService.GetOriginPoint internally checks if foregroundHwnd is 0 (desktop focused / system dialog) and falls back to the primary monitor center as origin. This satisfies the locked decision "When no foreground window can be determined, use screen center as origin." The originX/originY out-params will reflect the screen center in that case, so verbose logging will correctly show the fallback origin.

6. Verbose logging (only when _verbose is true): Log origin HWND, center coordinates, direction, and candidate count to `Console.Error.WriteLine` with `[HH:mm:ss.fff]` timestamp prefix matching Phase 7's pattern. Example format:
   ```
   [14:22:31.456] Navigate: left | origin: 0x000A0B2C center=(960, 540) | candidates: 2
   ```

7. Activate: `int result = FocusActivator.ActivateWithWrap(ranked, filtered, dir, config.Strategy, config.Wrap, _verbose);` — FocusActivator.ActivateWithWrap already handles the wrap behavior and iterates candidates on failure. Pass `_verbose` so FocusActivator logs activation attempts.

8. Verbose logging of result: If result == 0 and _verbose, log "Navigate: {direction} -> success". If result == 2 and _verbose, log warning "Navigate: {direction} -> all activations failed". If result == 1 (no candidates), do NOT log — silent no-op per user decision.

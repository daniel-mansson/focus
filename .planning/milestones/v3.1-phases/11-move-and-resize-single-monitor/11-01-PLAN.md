**Step 2: Create WindowManagerService.cs**

Create `focus/Windows/Daemon/WindowManagerService.cs` as a static class in the `Focus.Windows.Daemon` namespace. Use `[SupportedOSPlatform("windows6.0.6000")]` attribute (same as OverlayOrchestrator).

Required usings: `System.Runtime.InteropServices`, `System.Runtime.Versioning`, `Focus.Windows`, `Windows.Win32`, `Windows.Win32.Foundation`, `Windows.Win32.Graphics.Dwm`, `Windows.Win32.Graphics.Gdi`, `Windows.Win32.UI.WindowsAndMessaging` (for SET_WINDOW_POS_FLAGS).

**Public method:** `public static unsafe void MoveOrResize(string direction, WindowMode mode, bool verbose)`

Implementation flow (see 11-RESEARCH.md Pattern 2 for full code):
1. Get foreground HWND via `PInvoke.GetForegroundWindow()` -- return if default
2. Guard 1: `if (PInvoke.IsZoomed(fgHwnd)) return;` -- maximized window = silent no-op (per locked decision: refuse, do NOT restore)
3. Get monitor work area via `GetWorkArea(fgHwnd)` private helper (PInvoke.MonitorFromWindow + PInvoke.GetMonitorInfo + mi.rcWork, same as OverlayOrchestrator.ClampToMonitor pattern)
4. Load fresh config: `var config = FocusConfig.Load();`
5. Compute grid step: `GridCalculator.GetGridStep(workWidth, workHeight, config.GridFractionX, config.GridFractionY)`
6. Compute snap tolerance per axis: `GridCalculator.GetSnapTolerancePx(stepX, config.SnapTolerancePercent)` for both X and Y
7. Dual-rect pattern:
   - `PInvoke.GetWindowRect(fgHwnd, out winRect)` -- return if fails
   - `PInvoke.DwmGetWindowAttribute(fgHwnd, DWMWINDOWATTRIBUTE.DWMWA_EXTENDED_FRAME_BOUNDS, ...)` for visRect -- return if fails
   - Compute border offsets: `borderLeft = visRect.left - winRect.left`, etc. (see 11-RESEARCH.md Pattern 2)
8. Dispatch to ComputeMove/ComputeGrow/ComputeShrink based on mode (use switch expression)
9. Call `PInvoke.SetWindowPos(fgHwnd, default, newX, newY, newCx, newCy, SWP_NOZORDER | SWP_NOACTIVATE | SWP_NOOWNERZORDER)`
10. Guard 2: if `SetWindowPos` returns false and verbose, log "SetWindowPos failed (elevated window?)"

**Private helper: `GetWorkArea(HWND hwnd)`** -- returns RECT rcWork (see 11-RESEARCH.md Pattern 2 code).

**Private method: `ComputeMove`** (see 11-RESEARCH.md Pattern 3 for full code):
- Takes direction, visRect, winRect, workArea, stepX, stepY, tolX, tolY, borderL/T/R/B
- Snap-first on the movement axis ONLY (perpendicular axis unchanged):
  - For left/right: check `GridCalculator.IsAligned(vis.left, work.left, stepX, tolX)` -- if aligned, step by `sign * stepX`; if not, snap via `GridCalculator.NearestGridLine(vis.left, work.left, stepX)`
  - For up/down: same logic on Y axis using `vis.top`, `work.top`, `stepY`, `tolY`
- Clamp: `Math.Clamp(newVisLeft, work.left, work.right - visW)` for X, same pattern for Y (MOVE-03)
- Translate visible coords back to winRect space by subtracting/adding border offsets
- Return new RECT in GetWindowRect coordinate space

**Stub ComputeGrow and ComputeShrink** -- for now, return the unchanged `win` rect (Task 2 implements them). This ensures the code compiles and the Move operation can be tested independently.

**ComputeGrow** (see 11-RESEARCH.md Pattern 4 for full code):

The edge in the pressed direction moves OUTWARD. The opposite edge is FIXED. Per locked decision: growing stops at the monitor work area boundary; do NOT push the opposite edge when hitting the boundary.

- `direction == "right"`: Move right edge rightward
  - Check `GridCalculator.IsAligned(vis.right, work.left, stepX, tolX)` -- if aligned, `vis.right + stepX`; if not, snap via `NearestGridLine(vis.right, work.left, stepX)`
  - Clamp: `Math.Min(newVisRight, work.right)` (SIZE-04)
- `direction == "left"`: Move left edge leftward
  - Check `IsAligned(vis.left, work.left, stepX, tolX)` -- if aligned, `vis.left - stepX`; if not, snap
  - Clamp: `Math.Max(newVisLeft, work.left)` (SIZE-04)
- `direction == "down"`: Move bottom edge downward
  - Check `IsAligned(vis.bottom, work.top, stepY, tolY)` -- if aligned, `vis.bottom + stepY`; if not, snap
  - Clamp: `Math.Min(newVisBottom, work.bottom)` (SIZE-04)
- `direction == "up"`: Move top edge upward
  - Check `IsAligned(vis.top, work.top, stepY, tolY)` -- if aligned, `vis.top - stepY`; if not, snap
  - Clamp: `Math.Max(newVisTop, work.top)` (SIZE-04)

Translate visible coords back to winRect space using border offsets. Return new RECT.

**ComputeShrink** (see 11-RESEARCH.md Pattern 5 for full code):

The edge in the pressed direction moves INWARD. Per locked decision: shrink stops when the window dimension would go below one grid cell (1 grid step). At minimum size, the operation is a silent no-op.

- Before any calculation, check minimum size guard:
  - For left/right: `if (visW <= stepX) return win;` (return unchanged rect = no-op) (SIZE-03)
  - For up/down: `if (visH <= stepY) return win;` (SIZE-03)
- `direction == "right"`: Right edge moves leftward (shrink width)
  - Check `IsAligned(vis.right, work.left, stepX, tolX)` -- if aligned, `vis.right - stepX`; if not, snap
  - Min-size clamp: `Math.Max(newVisRight, newVisLeft + stepX)` (SIZE-03)
- `direction == "left"`: Left edge moves rightward (shrink width)
  - Check `IsAligned(vis.left, work.left, stepX, tolX)` -- if aligned, `vis.left + stepX`; if not, snap
  - Min-size clamp: `Math.Min(newVisLeft, newVisRight - stepX)` (SIZE-03)
- `direction == "down"`: Bottom edge moves upward (shrink height)
  - Check `IsAligned(vis.bottom, work.top, stepY, tolY)` -- if aligned, `vis.bottom - stepY`; if not, snap
  - Min-size clamp: `Math.Max(newVisBottom, newVisTop + stepY)` (SIZE-03)
- `direction == "up"`: Top edge moves downward (shrink height)
  - Check `IsAligned(vis.top, work.top, stepY, tolY)` -- if aligned, `vis.top + stepY`; if not, snap
  - Min-size clamp: `Math.Min(newVisTop, newVisBottom - stepY)` (SIZE-03)

Translate visible coords back to winRect space using border offsets. Return new RECT.

**Snap direction note (from research Pattern 6):** For grow/shrink, the snap check applies to the EDGE being moved, not the window origin. The snap origin is always `work.left` (for X) and `work.top` (for Y). Grid lines are at `work.origin + N*step`. When "snap only" triggers on a grow/shrink, the edge snaps to grid -- this IS a visible size change and is correct behavior per snap-first semantics.

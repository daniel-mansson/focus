Create `focus/Windows/Daemon/KeyboardHookHandler.cs` implementing the WH_KEYBOARD_LL global keyboard hook.

```csharp
using System.Runtime.InteropServices;
using System.Runtime.Versioning;
using System.Threading.Channels;
using Windows.Win32;
using Windows.Win32.Foundation;
using Windows.Win32.UI.WindowsAndMessaging;

namespace Focus.Windows.Daemon;

[SupportedOSPlatform("windows6.0.6000")]
internal sealed class KeyboardHookHandler : IDisposable { ... }
```

**CRITICAL REQUIREMENT — Static delegate field:**
Declare `private static HOOKPROC? s_hookProc;` as a **static** field. The GC WILL collect a local or instance delegate reference after the installing method returns, crashing the process with ExecutionEngineException. This is non-negotiable — see Pitfall 2 in research.

**Constructor:**
- Accept `ChannelWriter<KeyEvent> channelWriter` parameter
- Store in private field `_channelWriter`

**Install() method:**
- Assign `s_hookProc = HookCallback;` (static field)
- Get module handle: `var hmod = new FreeLibrarySafeHandle(PInvoke.GetModuleHandle((string?)null), false);` — the `false` parameter is critical (prevents FreeLibrary on dispose, see Pitfall 3 in research)
- Call `_hookHandle = PInvoke.SetWindowsHookEx(WINDOWS_HOOK_ID.WH_KEYBOARD_LL, s_hookProc, hmod, 0);`
- If `_hookHandle.IsInvalid`: throw `InvalidOperationException($"Failed to install keyboard hook: error {Marshal.GetLastWin32Error()}")` (user decision: print clear error, exit immediately)

**HookCallback(int nCode, WPARAM wParam, LPARAM lParam) method** — the HOOKPROC:
- Must be `private unsafe LRESULT HookCallback(int nCode, WPARAM wParam, LPARAM lParam)`
- If `nCode < 0`: return `PInvoke.CallNextHookEx(null, nCode, wParam, lParam)` immediately

- Cast lParam to struct: `var kbd = (KBDLLHOOKSTRUCT*)lParam.Value;`

- **DAEMON-06: Filter injected events** — if `((uint)kbd->flags & 0x00000010u) != 0` (LLKHF_INJECTED), return `CallNextHookEx` (pass through, do not process). This prevents AHK-synthesized keys from triggering CAPSLOCK detection.

- **Check for VK_CAPITAL (0x14):**
  - If `kbd->vkCode != 0x14`: return `CallNextHookEx` (pass through non-CAPSLOCK keys)

- **Filter modifier combinations (user decision: bare CAPSLOCK only):**
  - Check Alt: if `((uint)kbd->flags & 0x00000020u) != 0` (LLKHF_ALTDOWN), return `CallNextHookEx`
  - Check Ctrl: if `(PInvoke.GetKeyState(0x11) & 0x8000) != 0` (VK_CONTROL high bit set), return `CallNextHookEx`
  - Check Shift: if `(PInvoke.GetKeyState(0x10) & 0x8000) != 0` (VK_SHIFT high bit set), return `CallNextHookEx`

- **Post event to channel:**
  - Determine key direction: `bool isKeyDown = (uint)wParam.Value == 0x0100u || (uint)wParam.Value == 0x0104u;` (WM_KEYDOWN or WM_SYSKEYDOWN)
  - Fire-and-forget write: `_channelWriter.TryWrite(new KeyEvent(kbd->vkCode, isKeyDown, kbd->time));`
  - TryWrite on unbounded channel always succeeds — NEVER use WriteAsync in the hook callback (1000ms timeout, Pitfall 1)

- **Suppress CAPSLOCK toggle:** Return `(LRESULT)1` (non-zero = eat the key event). This prevents the CAPSLOCK toggle state from changing. Return this for ALL VK_CAPITAL events that pass the filters above (both key-down and key-up).

**Uninstall() method:**
- If `_hookHandle is { IsInvalid: false }`: call `PInvoke.UnhookWindowsHookEx(_hookHandle)`, set `_hookHandle = null`, set `s_hookProc = null` (allow GC)

**Dispose() method:**
- Call `Uninstall()`

**Type reference notes for CsWin32:**
- `HOOKPROC` is generated by CsWin32 from SetWindowsHookEx in NativeMethods.txt
- `KBDLLHOOKSTRUCT` is generated as a dependency of the hook APIs
- `FreeLibrarySafeHandle` is in `Microsoft.Win32.SafeHandles` (or `Windows.Win32` generated namespace — check which CsWin32 generates)
- Use `global::` prefix if `Windows.Win32` types conflict with `Focus.Windows` namespace
- GetKeyState returns `short` — check high bit with `& 0x8000`

Create `focus/Windows/Daemon/CapsLockMonitor.cs` implementing the CAPSLOCK hold/release state machine.

```csharp
using System.Threading.Channels;

namespace Focus.Windows.Daemon;

internal sealed class CapsLockMonitor { ... }
```

**Constructor:**
- Accept `ChannelReader<KeyEvent> reader` and `bool verbose` parameters
- Store in private fields

**RunAsync(CancellationToken cancellationToken) method:**
- This is the main consumer loop running on a worker thread (Task.Run)
- Use `await foreach (var evt in _reader.ReadAllAsync(cancellationToken))` to consume events
- Implement a simple hold/release state machine:
  - Track `_isHeld` boolean field (initially false)
  - On key-down event (`evt.IsKeyDown`):
    - If already held: ignore (repeat key events from Windows auto-repeat)
    - If not held: set `_isHeld = true`, call `OnCapsLockHeld()`
  - On key-up event (`!evt.IsKeyDown`):
    - If held: set `_isHeld = false`, call `OnCapsLockReleased()`
    - If not held: ignore (spurious release)

**OnCapsLockHeld() method:**
- If `_verbose`: write to stderr `$"[{DateTime.Now:HH:mm:ss.fff}] CAPSLOCK held"` (user decision: timestamped plain text to stderr)
- This is where Phase 6 will later hook overlay show logic. For now, debug logging only.

**OnCapsLockReleased() method:**
- If `_verbose`: write to stderr `$"[{DateTime.Now:HH:mm:ss.fff}] CAPSLOCK released"`
- This is where Phase 6 will later hook overlay hide logic. For now, debug logging only.

**ResetState() method (public):**
- Sets `_isHeld = false`
- Called after hook reinstall on wake (prevents stuck-held state after sleep/wake — see Pitfall 4 in research)

**Important:** The verbose log format uses `Console.Error.WriteLine` (stderr), not `Console.WriteLine` (stdout). This is consistent with v1's pattern where operational output goes to stderr and only structured output goes to stdout. The `[HH:mm:ss.fff]` timestamp format matches the user decision for timestamped plain text.

**1. OverlayWindow.cs** ‚Äî Create `focus/Windows/Daemon/Overlay/OverlayWindow.cs`:

A class that wraps a single Win32 layered window HWND. Manages full lifecycle: RegisterClassEx, CreateWindowEx, ShowWindow, SetWindowPos, DestroyWindow, UnregisterClass.

Key design:
- Namespace: `Focus.Windows.Daemon.Overlay`
- Class: `internal sealed class OverlayWindow : IDisposable`
- `[SupportedOSPlatform("windows6.0.6000")]`
- **Constructor** ‚Äî registers window class and creates the HWND:
  - Use `Marshal.GetHINSTANCE(typeof(OverlayWindow).Module)` for HINSTANCE (NOT `new HINSTANCE(0)` ‚Äî causes error 87)
  - Register with `PInvoke.RegisterClassEx`:
    - `lpszClassName`: unique class name per instance (e.g., `"FocusOverlay_" + Guid.NewGuid().ToString("N")[..8]`) ‚Äî store in a field for UnregisterClass
    - `lpfnWndProc`: a WNDPROC delegate ‚Äî MUST be stored in a static or instance field to prevent GC collection (same pattern as KeyboardHookHandler's HOOKPROC)
    - `hInstance`: the HINSTANCE from Marshal.GetHINSTANCE
    - Other fields: zeroes/defaults
  - Create window with `PInvoke.CreateWindowEx`:
    - `dwExStyle`: `WS_EX_LAYERED | WS_EX_TRANSPARENT | WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE | WS_EX_TOPMOST` (OVERLAY-02: click-through, excluded from Alt+Tab/taskbar, non-focus-stealing, always-on-top)
    - `dwStyle`: `WS_POPUP` (borderless, no caption)
    - Position/size: 0,0,0,0 (will be set by SetWindowPos later)
    - `hWndParent`: `HWND.Null`
    - `hInstance`: same HINSTANCE
  - Store the returned HWND in a field
  - Window starts hidden (not shown until Show is called)

- **WndProc** ‚Äî static or instance method matching WNDPROC signature:
  - Handle `WM_PAINT` (0x000F): call `PInvoke.BeginPaint` + `PInvoke.EndPaint` and return 0 ‚Äî CRITICAL to prevent infinite WM_PAINT loop and 100% CPU
  - All other messages: call `PInvoke.DefWindowProc`

- **Show(RECT bounds)** ‚Äî positions and shows the overlay:
  - Call `PInvoke.SetWindowPos(hwnd, HWND_TOPMOST, bounds.left, bounds.top, bounds.right - bounds.left, bounds.bottom - bounds.top, SWP_NOACTIVATE | SWP_SHOWWINDOW)` ‚Äî ALWAYS pass SWP_NOACTIVATE

- **Hide()** ‚Äî hides the overlay:
  - Call `PInvoke.ShowWindow(hwnd, SW_HIDE)`

- **Property `HWND Hwnd`** ‚Äî exposes the HWND for the renderer to call UpdateLayeredWindow

- **Dispose** ‚Äî cleanup:
  - Call `PInvoke.DestroyWindow(hwnd)` if HWND is valid
  - Call `PInvoke.UnregisterClass(className, hInstance)` to unregister the window class
  - Null out the WNDPROC delegate field

**CRITICAL anti-patterns to avoid:**
- Do NOT use `new HINSTANCE(0)` ‚Äî use `Marshal.GetHINSTANCE(typeof(OverlayWindow).Module)`
- Do NOT forget to handle WM_PAINT ‚Äî causes 100% CPU
- Do NOT forget SWP_NOACTIVATE on SetWindowPos ‚Äî causes focus steal
- The WNDPROC delegate MUST be stored in a field (instance or static) ‚Äî GC will collect a local variable after the method returns, and the callback pointer will become invalid
- Use `unsafe` code freely ‚Äî AllowUnsafeBlocks is already true in csproj

**CsWin32 type notes:**
- Window style constants: CsWin32 generates these as `WINDOW_STYLE` and `WINDOW_EX_STYLE` enum types. Use them from the `Windows.Win32.UI.WindowsAndMessaging` namespace.
- `HWND_TOPMOST`: may need to be created as `new HWND(-1)` if CsWin32 doesn't generate a constant.
- For `RegisterClassEx`, CsWin32 takes `WNDCLASSEXW*` ‚Äî use `fixed` or stackalloc.
- For string class name in `CreateWindowEx`, CsWin32 expects `PCWSTR` ‚Äî use `fixed (char* pClassName = className)`.

**2. BorderRenderer.cs** ‚Äî Create `focus/Windows/Daemon/Overlay/BorderRenderer.cs`:

The default renderer that draws colored borders with rounded corners using GDI into a premultiplied-alpha DIB.

Key design:
- Namespace: `Focus.Windows.Daemon.Overlay`
- Class: `internal sealed class BorderRenderer : IOverlayRenderer`
- `[SupportedOSPlatform("windows6.0.6000")]`
- Property `string Name => "border"` (RENDER-03, CFG-07)
- Constants:
  - `BorderThickness = 2` (user decision: 2-3px, default to 2 physical pixels)
  - `CornerEllipse = 16` (Win11 ~8px radius = 16px diameter for GDI RoundRect)

- **Paint(HWND hwnd, RECT bounds, uint argbColor)** implementation:
  1. Calculate width = `bounds.right - bounds.left`, height = `bounds.bottom - bounds.top`
  2. If width <= 0 or height <= 0, return early
  3. Extract ARGB components from `argbColor`: alpha = `(byte)(argbColor >> 24)`, R = `(byte)(argbColor >> 16)`, G = `(byte)(argbColor >> 8)`, B = `(byte)(argbColor)`
  4. Get screen DC: `var screenDC = PInvoke.GetDC(default)`
  5. Create 32bpp top-down DIB section:
     ```
     BITMAPINFO bmi = default;
     bmi.bmiHeader.biSize = (uint)sizeof(BITMAPINFOHEADER);
     bmi.bmiHeader.biWidth = width;
     bmi.bmiHeader.biHeight = -height;  // top-down
     bmi.bmiHeader.biPlanes = 1;
     bmi.bmiHeader.biBitCount = 32;
     bmi.bmiHeader.biCompression = 0; // BI_RGB
     void* bits;
     var hBitmap = PInvoke.CreateDIBSection(screenDC, &bmi, DIB_USAGE.DIB_RGB_COLORS, &bits, null, 0);
     ```
  6. Create compatible DC and select bitmap: `var memDC = PInvoke.CreateCompatibleDC(screenDC); var oldBitmap = PInvoke.SelectObject(memDC, hBitmap);`
  7. Clear the DIB to fully transparent (zero all bytes):
     ```
     PInvoke.GdiFlush();
     System.Runtime.InteropServices.NativeMemory.Clear(bits, (nuint)(width * height * 4));
     ```
  8. Set background mode transparent: `PInvoke.SetBkMode(memDC, BACKGROUND_MODE.TRANSPARENT)`
  9. Create GDI pen with the border color (GDI uses COLORREF = 0x00BBGGRR, no alpha):
     ```
     var colorRef = new COLORREF((uint)(B | (G << 8) | (R << 16)));
     var hPen = PInvoke.CreatePen(PEN_STYLE.PS_SOLID, BorderThickness, colorRef);
     ```
  10. Select NULL_BRUSH for hollow interior: `var hBrush = PInvoke.GetStockObject(GET_STOCK_OBJECT_FLAGS.NULL_BRUSH); var oldPen = PInvoke.SelectObject(memDC, hPen); var oldBrush = PInvoke.SelectObject(memDC, hBrush);`
  11. Draw rounded rectangle (border only): `PInvoke.RoundRect(memDC, 0, 0, width, height, CornerEllipse, CornerEllipse);`
  12. Apply premultiplied alpha to all GDI-drawn pixels:
      ```
      PInvoke.GdiFlush();
      uint* pixelBuf = (uint*)bits;
      for (int i = 0; i < width * height; i++)
      {
          uint pixel = pixelBuf[i];
          byte a = (byte)(pixel >> 24);
          if (a != 0)  // Only modify pixels GDI touched
          {
              byte pr = (byte)(((pixel >> 16) & 0xFF) * alpha / 255);
              byte pg = (byte)(((pixel >> 8) & 0xFF) * alpha / 255);
              byte pb = (byte)((pixel & 0xFF) * alpha / 255);
              pixelBuf[i] = ((uint)alpha << 24) | ((uint)pr << 16) | ((uint)pg << 8) | pb;
          }
      }
      ```
  13. Call UpdateLayeredWindow:
      ```
      var blend = new BLENDFUNCTION
      {
          BlendOp = 0,             // AC_SRC_OVER
          BlendFlags = 0,
          SourceConstantAlpha = 255,
          AlphaFormat = 1          // AC_SRC_ALPHA
      };
      var ptDst = new POINT { x = bounds.left, y = bounds.top };
      var sizeSrc = new SIZE { cx = width, cy = height };
      var ptSrc = new POINT { x = 0, y = 0 };
      PInvoke.UpdateLayeredWindow(hwnd, screenDC, &ptDst, &sizeSrc, memDC, &ptSrc, new COLORREF(0), &blend, UPDATE_LAYERED_WINDOW_FLAGS.ULW_ALPHA);
      ```
  14. Cleanup GDI resources in reverse order (CRITICAL to prevent GDI leaks):
      ```
      PInvoke.SelectObject(memDC, oldPen);
      PInvoke.SelectObject(memDC, oldBrush);
      PInvoke.DeleteObject(hPen);
      // Do NOT DeleteObject on NULL_BRUSH ‚Äî it is a stock object
      PInvoke.SelectObject(memDC, oldBitmap);
      PInvoke.DeleteDC(memDC);
      PInvoke.DeleteObject(hBitmap);
      PInvoke.ReleaseDC(default, screenDC);
      ```

**CsWin32 type resolution notes for the executor:**
- `BLENDFUNCTION` may live in `Windows.Win32.Graphics.Gdi` namespace
- `UPDATE_LAYERED_WINDOW_FLAGS` may be named differently in CsWin32 ‚Äî check generated code or use raw `(UPDATE_LAYERED_WINDOW_FLAGS)2` for `ULW_ALPHA`
- `DIB_USAGE`, `PEN_STYLE`, `BACKGROUND_MODE`, `GET_STOCK_OBJECT_FLAGS` ‚Äî these are CsWin32 enum types in the `Windows.Win32.Graphics.Gdi` namespace
- `COLORREF` ‚Äî CsWin32 generates this as a struct with a uint constructor
- If any type is not found after adding to NativeMethods.txt, check the CsWin32 generated files in `obj/Debug/net8.0-windows/generated/` for the exact namespace and name
- The `unsafe` keyword is needed for pointer operations (bits, pixelBuf, BITMAPINFO*, etc.)

**New methods to add (place after BuildKeybindingsGroup, before grid preview section):**

1. **`DetectTaskState()`** -- static method, returns `(bool exists, bool isElevated)`:
   - Use `Process.Start` with `UseShellExecute = false`, `RedirectStandardOutput = true`, `RedirectStandardError = true`, `CreateNoWindow = true`
   - Run `schtasks.exe /Query /TN "FocusDaemon" /XML`
   - Read StandardOutput.ReadToEnd(), then WaitForExit()
   - If exit code != 0 or exception: return (false, false)
   - Check if output contains "HighestAvailable" (StringComparison.Ordinal) for isElevated
   - Wrap entire method in try/catch returning (false, false) on any exception

2. **`RunSchtasksElevated(string arguments)`** -- static method, returns `bool`:
   - Use `Process.Start` with `UseShellExecute = true`, `Verb = "runas"`, `WindowStyle = ProcessWindowStyle.Hidden`
   - Run `schtasks.exe` with the given arguments
   - WaitForExit(), return `process.ExitCode == 0`
   - Catch `System.ComponentModel.Win32Exception` when `ex.NativeErrorCode == 1223` (UAC cancelled) -- return false
   - NOTE: UseShellExecute=true is REQUIRED for Verb="runas" -- do NOT set RedirectStandardOutput (mutually exclusive)

3. **`BuildTaskXml(string appPath, bool runElevated)`** -- static method, returns `string`:
   - Use C# raw string literal (triple-quote) to build XML identical to installer/focus.iss BuildTaskXml
   - Dynamic values: `{runLevel}` = runElevated ? "HighestAvailable" : "LeastPrivilege", `{appPath}` = the exe path
   - Must include: ONLOGON trigger, InteractiveToken, ExecutionTimeLimit=PT0S, Priority=7, `daemon --background` arguments
   - Match the installer's XML structure EXACTLY (same elements, same order, same values)

4. **`CreateTask(bool elevated)`** -- instance method, returns `bool`:
   - Get exe path via `Environment.ProcessPath!`
   - Build XML via `BuildTaskXml(exePath, elevated)`
   - Write XML to temp file: `Path.Combine(Path.GetTempPath(), "FocusDaemon.xml")`
   - Use `File.WriteAllText(xmlPath, xml, System.Text.Encoding.Unicode)` -- UTF-16 LE to match XML encoding declaration
   - Delete existing task first (ignore failure): `RunSchtasksElevated("/Delete /TN \"FocusDaemon\" /F")`
   - Create task: `RunSchtasksElevated($"/Create /XML \"{xmlPath}\" /TN \"FocusDaemon\" /F")`
   - Clean up temp file in finally block (ignore delete failure)
   - Return whether create succeeded

5. **`DeleteTask()`** -- static method, returns `bool`:
   - Try non-elevated first: `Process.Start` with `UseShellExecute = false`, `CreateNoWindow = true`, schtasks /Delete /TN "FocusDaemon" /F
   - If exit code == 0, return true
   - If non-elevated failed, fall back to `RunSchtasksElevated("/Delete /TN \"FocusDaemon\" /F")`
   - Wrap non-elevated attempt in try/catch (fall through to elevated on any exception)

**UI construction -- `BuildStartupGroup()` method:**
- Create GroupBox via `MakeGroup("Startup", 456, 100)` (adjust height as needed to fit two checkboxes + label)
- Add `_startupCheck` CheckBox: Text = "Run at startup", Location at (12, 24), AutoSize = true
- Add `_elevationCheck` CheckBox: Text = "Request elevated permissions", Location at (12, 48), AutoSize = true
- Add explanatory Label: Text = "Required to navigate between admin windows", Location at (30, 70), AutoSize = true, Font = new Font("Segoe UI", 8f), ForeColor = SystemColors.GrayText
- Call `DetectTaskState()` to get initial state BEFORE wiring event handlers
- Set `_startupCheck.Checked = exists` and `_elevationCheck.Checked = isElevated` and `_elevationCheck.Enabled = exists`
- THEN wire event handlers: `_startupCheck.CheckedChanged += OnStartupToggled` and `_elevationCheck.CheckedChanged += OnElevationToggled`
- This order prevents triggering schtasks operations during form initialization

**Event handlers:**

6. **`OnStartupToggled(object? sender, EventArgs e)`** -- async void:
   - Capture `bool wantStartup = _startupCheck.Checked`
   - Disable both checkboxes (_startupCheck.Enabled = false, _elevationCheck.Enabled = false)
   - Run schtasks on background thread: `bool success = await Task.Run(() => { ... })`
     - If wantStartup: call `CreateTask(elevated: _elevationCheck.Checked)`
     - If !wantStartup: call `DeleteTask()`
   - If !success: silently revert toggle -- MUST unhook handler first to prevent recursion:
     `_startupCheck.CheckedChanged -= OnStartupToggled; _startupCheck.Checked = !wantStartup; _startupCheck.CheckedChanged += OnStartupToggled;`
   - Update dependent controls:
     - `_elevationCheck.Enabled = _startupCheck.Checked`
     - If !_startupCheck.Checked: unhook _elevationCheck handler, set Checked = false, rehook
   - Re-enable _startupCheck: `_startupCheck.Enabled = true`
   - CONTEXT.md says "Unexpected error (schtasks failure): revert toggle + show MessageBox with error details". To distinguish UAC cancel from schtasks failure, modify RunSchtasksElevated to use a tri-state or exception-based approach. Simplest: have CreateTask/DeleteTask throw on schtasks failure but return false on UAC cancel. Then OnStartupToggled can catch exceptions for MessageBox. Implementation detail left to executor's judgment.

7. **`OnElevationToggled(object? sender, EventArgs e)`** -- async void:
   - Capture `bool wantElevated = _elevationCheck.Checked`
   - Disable both checkboxes
   - Recreate task with new RunLevel: `bool success = await Task.Run(() => CreateTask(elevated: wantElevated))`
   - If !success: silently revert toggle (unhook, set, rehook pattern)
   - Re-enable both checkboxes

**Wire into BuildUi():**
- After `root.Controls.Add(BuildKeybindingsGroup());` add `root.Controls.Add(BuildStartupGroup());`
- Change `ClientSize = new Size(500, 700)` to `ClientSize = new Size(500, 780)`

**Anti-patterns to avoid (from RESEARCH.md):**
- Do NOT store startup state in config.json -- always query schtasks
- Do NOT use UseShellExecute=false with Verb="runas" -- runas is silently ignored
- Do NOT use UseShellExecute=true with RedirectStandardOutput -- throws InvalidOperationException
- Do NOT call WaitForExit on UI thread without Task.Run -- freezes UI during UAC
- Do NOT set Checked programmatically without unhooking handler first -- causes recursion

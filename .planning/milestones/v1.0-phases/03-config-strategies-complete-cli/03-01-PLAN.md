- Same structure as ScoreCandidate (nearest-edge distance, directional filter, primary/secondary decomposition).
- Same strict inequality directional filter as ScoreCandidate.
- Uses `primaryWeight = 1.0` and `secondaryWeight = 5.0` (vs balanced's 2.0) — per research recommendation (Claude's discretion).
- Comment: "Higher secondary weight = more aggressive lane preference vs balanced's 2.0"

**2. Add ScoreClosestInDirection method** (after ScoreStrongAxisBias):
```csharp
internal static double ScoreClosestInDirection(
    double originX, double originY,
    WindowInfo candidate,
    Direction direction)
```
- Compute candidate CENTER: `candCx = (candidate.Left + candidate.Right) / 2.0`, `candCy = (candidate.Top + candidate.Bottom) / 2.0`.
- Half-plane cone check on center (not nearest edge): `dx = candCx - originX`, `dy = candCy - originY`. Direction.Left → `dx < 0`, Right → `dx > 0`, Up → `dy < 0`, Down → `dy > 0`. If not in cone → `double.MaxValue`.
- Score = `Math.Sqrt(dx * dx + dy * dy)` — pure Euclidean center-to-center distance.
- Comment: "Pure distance with wide cone (~90° half-plane) — picks nearest window center-to-center"
- Note: This uses center-to-center (not nearest-edge) per locked decision.

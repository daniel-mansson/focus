<objective>
Build the directional scoring engine: Direction enum, NavigationService with balanced scoring algorithm, and primary monitor fallback -- the core geometry that decides which window to navigate to.

Purpose: This is the algorithmic core of the tool. Given a direction and the set of visible windows from Phase 1's WindowEnumerator, NavigationService scores every candidate by primary-axis distance plus perpendicular penalty, filters to only in-direction windows, and returns a ranked list. Plan 02-02 will consume this ranked list to activate the best window.

Output: Direction.cs (enum + parser), NavigationService.cs (scoring + ranking), updated MonitorHelper.cs (primary monitor center), updated NativeMethods.txt (new Win32 APIs for Phase 2).

**MonitorHelper.cs:** Add a `GetPrimaryMonitorCenter` static method to the existing MonitorHelper class. This provides the fallback origin when GetForegroundWindow returns NULL (desktop focused):

```csharp
/// <summary>
/// Returns the center point of the primary monitor in virtual screen coordinates.
/// Used as origin fallback when no foreground window exists.
/// </summary>
public static unsafe (double X, double Y) GetPrimaryMonitorCenter()
{
    // MONITORINFOF_PRIMARY not generated by CsWin32 (issue #1004) -- use constant
    HMONITOR hm = PInvoke.MonitorFromPoint(default, MONITOR_FROM_FLAGS.MONITOR_DEFAULTTOPRIMARY);
    MONITORINFO mi = default;
    mi.cbSize = (uint)sizeof(MONITORINFO);
    if (PInvoke.GetMonitorInfo(hm, ref mi))
    {
        return (
            (mi.rcMonitor.left + mi.rcMonitor.right) / 2.0,
            (mi.rcMonitor.top + mi.rcMonitor.bottom) / 2.0
        );
    }
    return (0.0, 0.0); // ultimate fallback
}
```

Add `using Windows.Win32.UI.WindowsAndMessaging;` if not already present (needed for MONITORINFO -- note: MonitorHelper currently imports from `Windows.Win32.Graphics.Gdi` and `Windows.Win32.Foundation`, check if MONITORINFO is in the Gdi namespace via the generated code). **NavigationService.cs:** Create `focus/Windows/NavigationService.cs` with the following structure. Use `[SupportedOSPlatform("windows5.0")]` at class level per established pattern.

The class must implement these methods:

1. **`GetRankedCandidates(List<WindowInfo> allWindows, Direction direction)`** -- the main entry point:
   - Call `PInvoke.GetForegroundWindow()` to get the current foreground HWND
   - Call `GetOriginPoint(foregroundHwnd)` to get the (originX, originY) center
   - Filter out the foreground window from candidates (locked decision: always exclude focused window)
   - Score each remaining window with `ScoreCandidate(originX, originY, window, direction)`
   - Filter out eliminated candidates (score == double.MaxValue)
   - Sort by score ascending (lowest = best)
   - Return `List<(WindowInfo Window, double Score)>`

2. **`GetOriginPoint(nint foregroundHwnd)`** -- determines the scoring origin:
   - If foregroundHwnd != 0: get bounds via `DwmGetWindowAttribute(DWMWA_EXTENDED_FRAME_BOUNDS)` and return geometric center
   - If foregroundHwnd == 0 OR bounds retrieval fails: fall back to `MonitorHelper.GetPrimaryMonitorCenter()`
   - Use the established `MemoryMarshal.AsBytes(MemoryMarshal.CreateSpan(ref boundsRect, 1))` pattern for the DWM call

3. **`NearestPoint(double px, double py, int left, int top, int right, int bottom)`** -- nearest point on rectangle:
   - `Math.Clamp(px, left, right)` for X, `Math.Clamp(py, top, bottom)` for Y
   - Returns `(double NearX, double NearY)`

4. **`ScoreCandidate(double originX, double originY, WindowInfo candidate, Direction direction)`** -- the balanced scoring formula:
   - Compute nearest point on candidate bounds from origin
   - **Directional filter:** The nearest point must be strictly in the requested direction from origin. Use strict inequality (`<` / `>`, not `<=` / `>=`) -- a window whose nearest edge is exactly at the origin line is ambiguous and should not match. Return `double.MaxValue` if not in direction.
   - **Primary axis distance:** How far in the requested direction (positive value)
     - Left: `originX - nearX`
     - Right: `nearX - originX`
     - Up: `originY - nearY`
     - Down: `nearY - originY`
   - **Secondary axis distance:** Perpendicular deviation (absolute value)
     - Left/Right: `Math.Abs(nearY - originY)`
     - Up/Down: `Math.Abs(nearX - originX)`
   - **Score formula (balanced strategy, NAV-07):**
     `primaryWeight * primaryDist + secondaryWeight * secondaryDist`
     Use `const double primaryWeight = 1.0; const double secondaryWeight = 2.0;`
     (Claude's discretion: 2.0 secondary weight makes alignment matter -- a window directly ahead beats a diagonally offset one. These are tunable constants.)
   - **Tie-breaking (Claude's discretion):** When two candidates have scores within epsilon (1e-6), prefer the one with smaller secondary distance (more aligned with the movement axis). If still tied, prefer the one with smaller primary distance (closer).

Mark `GetRankedCandidates` and `GetOriginPoint` as `public static` (called from Program.cs in Plan 02-02). Mark `ScoreCandidate` and `NearestPoint` as `internal static` (testable but not part of public API).

The class uses `unsafe` context for CsWin32 HWND conversions. Follow the Phase 1 pattern: `new HWND((void*)(IntPtr)hwndNint)` for nint-to-HWND conversion.

Create `focus/Windows/FocusActivator.cs` in the `Focus.Windows` namespace with `[SupportedOSPlatform("windows5.0")]` at class level.

**CRITICAL:** Before writing any code, read the 02-01-SUMMARY.md to get the exact CsWin32-generated field names for the INPUT struct. The research predicts `INPUT._Anonymous_e__Union` with `.ki` for KEYBDINPUT, but the actual names MUST come from the generated code inspection documented in Plan 02-01's summary. If the summary documents different names, use those instead of what's written below.

The class has two methods:

1. **`TryActivateWindow(nint hwnd)`** -- attempts to activate a single window:
   - Define `const ushort VK_MENU = 0x12;` (ALT virtual key -- not generated by CsWin32)
   - Build `Span<INPUT> inputs = stackalloc INPUT[2]`:
     - `inputs[0]`: type = `INPUT_TYPE.INPUT_KEYBOARD`, keyboard union member with `wVk = VK_MENU`, `dwFlags = 0` (key down)
     - `inputs[1]`: type = `INPUT_TYPE.INPUT_KEYBOARD`, keyboard union member with `wVk = VK_MENU`, `dwFlags = KEYBD_EVENT_FLAGS.KEYEVENTF_KEYUP`
   - Call `PInvoke.SendInput(inputs, Marshal.SizeOf<INPUT>())` -- CsWin32 generates a Span overload
   - Immediately (same thread, no delay) call `PInvoke.SetForegroundWindow(targetHwnd)`
   - Return `bool` -- true if SetForegroundWindow succeeded
   - Convert nint to HWND using established pattern: `new HWND((void*)(IntPtr)hwnd)`
   - Mark as `public static unsafe`

2. **`ActivateBestCandidate(List<(WindowInfo Window, double Score)> rankedCandidates)`** -- tries candidates in order:
   - If `rankedCandidates.Count == 0` return `1` (exit code: no candidates)
   - Iterate through candidates in order (already sorted by score, best first)
   - Call `TryActivateWindow(candidate.Window.Hwnd)` for each
   - If activation succeeds, return `0` (exit code: success)
   - If activation fails, silently continue to next candidate (locked decision: skip elevated windows)
   - If ALL candidates fail, return `2` (exit code: error -- candidates existed but none could be activated)
   - Mark as `public static`

Required using statements:
```csharp
using System.Runtime.InteropServices;
using System.Runtime.Versioning;
using Windows.Win32;
using Windows.Win32.Foundation;
using Windows.Win32.UI.Input.KeyboardAndMouse;
```

Note: The `INPUT_TYPE` and `KEYBD_EVENT_FLAGS` enums may be in `Windows.Win32.UI.Input.KeyboardAndMouse` namespace. Check the generated code paths if the build fails -- add the correct using statement.

**Anti-patterns to avoid:**
- Do NOT use `AttachThreadInput` -- reentrancy/deadlock risk
- Do NOT add any `Thread.Sleep` or delay between SendInput and SetForegroundWindow
- Do NOT use `keybd_event` -- it is legacy; `SendInput` is the supported API (locked in STATE.md)

   ```
   static int RunNavigation(string directionStr)
   {
       // Parse direction
       var direction = DirectionParser.Parse(directionStr);
       if (direction is null)
       {
           Console.Error.WriteLine($"Error: Unknown direction '{directionStr}'. Use: left, right, up, down");
           return 2;
       }

       try
       {
           // Enumerate windows (Phase 1 pipeline)
           var enumerator = new WindowEnumerator();

           // Activate best candidate (this plan)
           return FocusActivator.ActivateBestCandidate(ranked);
       }
       catch (Exception ex)
       {
           Console.Error.WriteLine($"Error: {ex.Message}");
           return 2;
       }
   }
   ```

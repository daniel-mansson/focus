  <action>
Create a standalone C# console project at `tools/generate-icon/` that generates the Focus icon and writes it to `focus/focus.ico`.

**Generator project (`tools/generate-icon/generate-icon.csproj`):**
- Target `net8.0-windows`, enable `UseWindowsForms` (for System.Drawing), OutputType `Exe`.
- No NuGet dependencies needed — uses only BCL and System.Drawing.

**Generator program (`tools/generate-icon/Program.cs`):**

1. **Icon drawing function** — `static byte[] DrawFocusIcon(int size)`:
   - Create `Bitmap(size, size, PixelFormat.Format32bppArgb)`, clear to `Color.Transparent`.
   - Set `SmoothingMode.None` — pixel-perfect geometric shapes, no anti-aliasing (anti-aliasing causes muddy gray fringe at small sizes per research).
   - Design parameters scaled to `size`:
     - `margin = Math.Max(1, size / 8)` — outer margin
     - `armLen = Math.Max(2, size / 4)` — bracket arm length
     - `thick = Math.Max(1, size / 12)` — line thickness (1px at 16, 2px at 24+)
     - `dotSize = Math.Max(1, size / 10)` — center dot half-width
   - Draw 4 L-shaped corner brackets in `Color.White`:
     - Top-left: horizontal arm `(margin, margin) -> (margin+armLen, margin)`, vertical arm `(margin, margin) -> (margin, margin+armLen)`
     - Top-right: mirror horizontally (`size-1-margin` as right edge)
     - Bottom-left: mirror vertically (`size-1-margin` as bottom edge)
     - Bottom-right: mirror both
   - Draw center dot: `FillRectangle` centered at `(size/2, size/2)` with size `dotSize*2+1`.
   - Save to `MemoryStream` as PNG (`ImageFormat.Png`), return `ms.ToArray()`.

2. **ICO encoder function** — `static void WriteIco(Stream output, IReadOnlyList<(byte[] PngData, int Width, int Height)> frames)`:
   - Write 6-byte ICO header: reserved=0, type=1 (ICO), count=frames.Count.
   - Write 16-byte directory entry per frame: width, height, colorCount=0, reserved=0, planes=1, bpp=32, dataSize=png.Length, dataOffset (calculated: 6 + 16*count + sum of preceding PNG sizes).
   - Write raw PNG bytes for each frame.
   - Use `BinaryWriter(output, Encoding.UTF8, leaveOpen: true)` to avoid closing the stream.
   - See 13-RESEARCH.md Pattern 1 for exact byte layout.

3. **Main function:**
   - Generate 4 sizes: 16, 20, 24, 32 (covers 100%, 125%, 150%, 200% DPI per research).
   - Call `DrawFocusIcon` for each size.
   - Call `WriteIco` to write `focus/focus.ico` (resolve path relative to the repo root — the generator runs from `tools/generate-icon/`, so output path is `../../focus/focus.ico`).
   - Print confirmation message to stdout.

**Run the generator:**
```bash
cd tools/generate-icon && dotnet run
```
This produces `focus/focus.ico`. The file is committed to the repo as a build input.

**Important:**
- Do NOT use `SmoothingMode.AntiAlias` — causes gray fringe on small icons.
- Do NOT use `Bitmap.Save(stream, ImageFormat.Icon)` — does NOT produce valid ICO format.
- Do NOT scale a single bitmap — draw each size independently for pixel-perfect results.

**Update `focus/focus.csproj`:**

1. Add `<ApplicationIcon>focus.ico</ApplicationIcon>` inside the existing `<PropertyGroup>` (after the `<UseWindowsForms>` line). This embeds the icon as a Win32 resource in the PE header — Explorer, taskbar, and Task Manager will display it.

2. Add an `<EmbeddedResource>` item in the existing `<ItemGroup>` (or a new one):
```xml
<ItemGroup>
  <EmbeddedResource Include="focus.ico">
    <LogicalName>focus.ico</LogicalName>
  </EmbeddedResource>
</ItemGroup>
```
Using `<LogicalName>focus.ico</LogicalName>` makes the manifest resource name exactly `"focus.ico"` — no namespace prefix guessing needed (per research Pitfall 2 mitigation).

3. Load the icon from the embedded resource:
```csharp
var iconStream = Assembly.GetExecutingAssembly()
```

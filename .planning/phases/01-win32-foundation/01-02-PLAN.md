  <action>
Create `focus/Windows/WindowEnumerator.cs` in namespace `Focus.Windows`:

**Class: WindowEnumerator**

Public method: `(List<WindowInfo> Windows, int FilteredUwpCount) GetNavigableWindows()`

Returns a tuple of the window list and the count of suppressed UWP duplicate HWNDs.

**Implementation steps (in order):**

1. **Enumerate monitors first** — call `MonitorHelper.EnumerateMonitors()` once and reuse the list for all windows.

2. **EnumWindows pass** — call `PInvoke.EnumWindows` with a callback. For each HWND, collect it into a raw list. Store the callback delegate in a local variable to prevent GC collection (RESEARCH.md Pitfall 2). The callback should collect ALL HWNDs; filtering happens in the next step.

3. **Alt+Tab filter** — for each raw HWND, apply the `IsAltTabWindow` check. This MUST replicate the exact Raymond Chen algorithm from RESEARCH.md Pattern 3:
   a. `PInvoke.IsWindowVisible(hwnd)` — skip if false (ENUM-02)
   b. `DwmGetWindowAttribute(DWMWA_CLOAKED)` — skip if `cloaked != 0` using `uint` not `BOOL` (ENUM-03, Pitfall 1)
   c. `PInvoke.IsIconic(hwnd)` — skip if true (ENUM-04)
   d. Read `GWL_EXSTYLE` via `GetWindowLongPtr` — extract `WS_EX_TOOLWINDOW` and `WS_EX_APPWINDOW` flags
   e. If `WS_EX_APPWINDOW` is set → include (overrides owner chain)
   f. Walk owner chain: `GetAncestor(GA_ROOTOWNER)`, then loop `GetLastActivePopup` until stable or visible
   g. If walk result != original HWND → skip
   h. If `WS_EX_TOOLWINDOW` → skip
   i. Otherwise → include

4. **UWP dedup** — after Alt+Tab filtering:
   a. For each passing HWND, get its class name via `PInvoke.GetClassName`
   b. If class is `"Windows.UI.Core.CoreWindow"` — check if a matching `ApplicationFrameWindow` parent is already in the result set. If so, suppress this HWND and increment `filteredUwpCount`. If not, keep it.
   c. If class is `"ApplicationFrameWindow"` — mark `IsUwpFrame = true` on the WindowInfo. For process name, find the child HWND with a different PID (RESEARCH.md Pattern 5) and use THAT child's process name instead of "ApplicationFrameHost.exe" (Pitfall 7).

5. **Build WindowInfo** for each surviving HWND:
   a. **Process name**: Call `GetWindowThreadProcessId` to get PID, then `OpenProcess(PROCESS_QUERY_LIMITED_INFORMATION)`, then `QueryFullProcessImageName`, then `Path.GetFileName()`. Close handle with `CloseHandle`. Return "?" if any step fails. For UWP frames, use the child process PID as described above.
   b. **Title**: Call `PInvoke.GetWindowText` (CsWin32 generated version, which uses GetWindowTextW internally). Use the CsWin32-generated overload that accepts a buffer.
   c. **Bounds**: Call `DwmGetWindowAttribute(DWMWA_EXTENDED_FRAME_BOUNDS)` into a RECT. Extract Left, Top, Right, Bottom. If the call fails, fall back to 0,0,0,0.
   d. **Monitor index**: Call `MonitorHelper.GetMonitorIndex(hwnd, monitors)`.
   e. **IsTopmost**: Check if `(exStyle & WS_EX_TOPMOST) != 0` from the extended style already read in the Alt+Tab filter.
   f. **IsUwpFrame**: Set true if class name is `"ApplicationFrameWindow"`.

6. **Return** the list (preserving EnumWindows Z-order, topmost first per CONTEXT.md) and the filtered UWP count.

**Important implementation notes:**
- All methods touching Win32 APIs will likely need `unsafe` context due to CsWin32 pointer-based signatures
- After adding `EnumWindows` to NativeMethods.txt (done in Plan 01), inspect the generated code at `obj/Debug/net9.0/Generated/` to understand the exact WNDENUMPROC signature before writing the callback. Adapt the callback syntax accordingly (it may be a delegate, a function pointer, or a generated overload).
- Use `Windows.Win32.PInvoke` as the namespace for CsWin32-generated calls
- Cast between CsWin32 types (HWND, HMONITOR, HANDLE) and nint as needed

Run `dotnet build` to verify compilation. Inspect any CsWin32-related errors carefully — they often indicate incorrect type usage or missing unsafe context.

  1. Instantiate `WindowEnumerator`
  2. Call `GetNavigableWindows()` to get `(windows, filteredUwpCount)`
  3. Call a local method or static helper to print the table
  4. Return 0

**Columnar table output** (locked decisions from CONTEXT.md):

Format: Aligned columnar table like `tasklist` / `docker ps`.

Columns (exact names and approximate widths — use your judgment for exact spacing):
- `HWND` (~12 chars) — display in hex format like `0x00031234` (planner's discretion: hex chosen for familiarity with Win32 tooling)
- `PROCESS` (~22 chars) — e.g. "chrome.exe"
- `TITLE` (~42 chars) — truncated at ~40 chars with `WindowInfo.TruncateTitle()`
- `BOUNDS` (~24 chars) — formatted as `L,T,R,B` e.g. "100,200,1400,900"
- `MON` (~4 chars) — 1-based monitor index
- `FLAGS` (~6 chars) — single-character markers, "T" for topmost

Print header row, then a separator line of dashes, then one row per window.

**Summary line** (after all rows):
- Format: `"Found {N} windows on {M} monitors"`
- If `filteredUwpCount > 0`, append: `" (filtered {K} duplicate UWP HWNDs)"`
- Print a blank line before the summary for readability

**Edge cases:**
- If no windows found: print "No navigable windows found." and return 0
- If WindowEnumerator throws an exception: catch, print error to stderr, return 2

Add `using Focus.Windows;` at the top of Program.cs.

Run `dotnet build` and then `dotnet run -- --debug enumerate` to verify the output looks correct. The output should show real windows currently open on the system.

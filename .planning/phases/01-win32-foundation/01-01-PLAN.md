**focus/Windows/WindowInfo.cs:**
Create a C# record (or readonly record struct) named `WindowInfo` in namespace `Focus.Windows`:
- `nint Hwnd` — raw HWND value (nint for CsWin32 interop)
- `string ProcessName` — e.g. "chrome.exe" (empty string if unavailable, not null)
- `string Title` — window title text (empty string if unavailable, not null)
- `int Left`, `int Top`, `int Right`, `int Bottom` — bounds from DWMWA_EXTENDED_FRAME_BOUNDS (physical pixels, L,T,R,B per CONTEXT.md locked decision)
- `int MonitorIndex` — 1-based monitor index
- `bool IsTopmost` — WS_EX_TOPMOST flag (for "T" flag column in debug output)
- `bool IsUwpFrame` — true if this HWND is an ApplicationFrameWindow container (for UWP dedup tracking)

Include a `string FlagsString` computed property that returns flag characters: "T" if IsTopmost, "" otherwise. This is extensible for future flags.

Include a static method `string TruncateTitle(string title, int maxLen = 40)` that returns `title[..(maxLen-3)] + "..."` if title.Length > maxLen, else returns title as-is. Handle edge case where title.Length <= 3 (return title unchanged).

**focus/Windows/MonitorHelper.cs:**
Create a static class `MonitorHelper` in namespace `Focus.Windows`:

1. `static List<nint> EnumerateMonitors()` — calls `PInvoke.EnumDisplayMonitors` with null HDC and null RECT to enumerate all monitors. Collects each HMONITOR handle as nint into a list and returns it. Must store the callback delegate in a local variable to prevent GC collection during enumeration.

2. `static int GetMonitorIndex(nint hwnd, List<nint> monitors)` — calls `PInvoke.MonitorFromWindow` with `MONITOR_DEFAULTTONEAREST` flag, finds the returned HMONITOR in the monitors list, returns `index + 1` (1-based). If not found, returns 1 as fallback.

Use CsWin32 generated types: cast between `HMONITOR` and `nint` as needed. The method signatures use `nint` for clean public API but internally work with CsWin32's `HWND`/`HMONITOR` types.

Mark methods as `unsafe` if required by CsWin32's generated EnumDisplayMonitors signature (it likely uses function pointers or unsafe callbacks).

Run `dotnet build` to verify everything still compiles after adding these files.

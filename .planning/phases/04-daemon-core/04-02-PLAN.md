```csharp
using System.Runtime.Versioning;
using System.Windows.Forms;
using System.Drawing;

namespace Focus.Windows.Daemon;

[SupportedOSPlatform("windows6.0.6000")]
internal sealed class DaemonApplicationContext : ApplicationContext { ... }
```

**Constructor accepts:**
- `KeyboardHookHandler hook` — for hook install (on start) and reinstall (on wake)
- `CapsLockMonitor monitor` — for state reset on wake
- `Action onExit` — callback invoked when user clicks Exit in tray (signals the daemon to shut down)

**Constructor body:**
1. Store parameters in private fields
2. Install the keyboard hook: `_hook.Install()` — this is safe because the message pump starts immediately after the constructor returns when `Application.Run(context)` is called. The hook callback requires a message pump, and by installing here, the hook is ready as soon as the pump starts.
3. Create the tray icon:
   ```csharp
   var menu = new ContextMenuStrip();
   menu.Items.Add("Exit", null, OnExitClicked);

   _trayIcon = new NotifyIcon
   {
       Icon = SystemIcons.Application,  // Built-in system icon (no embedded resource needed)
       ContextMenuStrip = menu,
       Text = "Focus Daemon",
       Visible = true
   };
   ```

**OnExitClicked handler:**
- Set `_trayIcon.Visible = false` (prevent ghost icon in tray)
- Call `_onExit()` callback (signals DaemonCommand to begin teardown)
- Call `Application.ExitThread()` (exits the message pump, which unblocks the STA thread)

**WM_POWERBROADCAST handling for sleep/wake recovery (user decision):**

Override WndProc is not directly available on ApplicationContext. Use a hidden NativeWindow to receive the message:

Create a private inner class `PowerBroadcastWindow : NativeWindow`:
- In its constructor: call `CreateHandle(new CreateParams())` to create a message-only window
- Override `WndProc(ref Message m)`:
  - If `m.Msg == 0x0218` (WM_POWERBROADCAST) AND `m.WParam == 0x0012` (PBT_APMRESUMEAUTOMATIC):
    - Call the reinstall callback: `_hook.Uninstall()` then `_hook.Install()` then `_monitor.ResetState()`
    - This handles hook recovery after system sleep — the hook may be invalidated by the OS
  - Always call `base.WndProc(ref m)`

Create an instance of `PowerBroadcastWindow` in the DaemonApplicationContext constructor and store it in a field.

**Dispose override:**
```csharp
protected override void Dispose(bool disposing)
{
    if (disposing)
    {
        _trayIcon?.Visible = false;
        _trayIcon?.Dispose();
        _powerWindow?.DestroyHandle();
    }
    base.Dispose(disposing);
}
```

**Important notes:**
- Do NOT call `_hook.Uninstall()` in Dispose — DaemonCommand handles cleanup order
- The tray icon uses `SystemIcons.Application` — no custom icon needed (Claude's discretion: system icon is sufficient for a developer tool)

**Part A: Create focus/Windows/Daemon/DaemonCommand.cs**

This is the top-level orchestrator that connects all daemon components.

```csharp
using System.Runtime.Versioning;
using System.Threading.Channels;
using System.Windows.Forms;

namespace Focus.Windows.Daemon;

[SupportedOSPlatform("windows6.0.6000")]
internal static class DaemonCommand { ... }
```

**Run(bool background, bool verbose) method returning int (exit code):**

The lifecycle sequence:

1. **Acquire single-instance mutex (DAEMON-04):**
   ```csharp
   Mutex? mutex = null;
   try
   {
       mutex = DaemonMutex.AcquireOrReplace();
   }
   catch (Exception ex)
   {
       Console.Error.WriteLine($"Error acquiring daemon mutex: {ex.Message}");
       return 2;
   }
   ```

2. **Print startup confirmation (user decision: one-line then silent):**
   ```csharp
   Console.WriteLine("Focus daemon started. Listening for CAPSLOCK.");
   ```

3. **Detach console if background mode (user decision: --background flag):**
   ```csharp
   if (background)
       PInvoke.FreeConsole();
   ```
   Print the confirmation BEFORE FreeConsole — after detach, Console writes may fail silently.

4. **Create the event channel:**
   ```csharp
   var channel = Channel.CreateUnbounded<KeyEvent>(new UnboundedChannelOptions
   {
       SingleWriter = true,
       SingleReader = true,
       AllowSynchronousContinuations = false  // Don't run continuations on hook thread
   });
   ```

5. **Create components:**
   ```csharp
   var hook = new KeyboardHookHandler(channel.Writer);
   var monitor = new CapsLockMonitor(channel.Reader, verbose);
   ```

6. **Set up cancellation:**
   ```csharp
   using var cts = new CancellationTokenSource();
   Console.CancelKeyPress += (_, e) =>
   {
       e.Cancel = true;  // Prevent immediate termination — allow cleanup
       cts.Cancel();
   };
   ```

7. **Start consumer task on thread pool:**
   ```csharp
   var consumerTask = Task.Run(() => monitor.RunAsync(cts.Token));
   ```

8. **Run STA message pump thread (this is where the hook lives):**
   ```csharp
   var staThread = new Thread(() =>
   {
       Application.Run(new DaemonApplicationContext(hook, monitor, () => cts.Cancel()));
   });
   staThread.SetApartmentState(ApartmentState.STA);
   staThread.IsBackground = false;  // Keep process alive
   staThread.Start();
   ```

9. **Wait for shutdown signal:**
   ```csharp
   // Block main thread until cancellation (Ctrl+C or tray Exit)
   try
   {
       cts.Token.WaitHandle.WaitOne();
   }
   catch (ObjectDisposedException) { }
   ```

10. **Cleanup (DAEMON-05: no orphaned hooks):**
    ```csharp
    // Signal the STA thread to exit
    if (staThread.IsAlive)
    {
        // Post quit message to the STA thread's message pump
        // Application.ExitThread() must be called from the STA thread
        // If tray Exit was clicked, it already called ExitThread
        // For Ctrl+C, we need to invoke on the STA thread:
        // Use a simple approach: the ApplicationContext handles this
    }

    // Unhook keyboard (safe to call multiple times)
    hook.Uninstall();
    hook.Dispose();

    // Signal channel completion so consumer loop exits
    channel.Writer.Complete();

    // Wait for consumer to finish (with timeout)
    try
    {
        consumerTask.Wait(TimeSpan.FromSeconds(3));
    }
    catch { }

    // Release mutex
    DaemonMutex.Release(mutex);
    ```

    For the STA thread shutdown when Ctrl+C is pressed (not tray Exit): The CancellationToken callback should also post `Application.ExitThread()` to the STA thread. One approach: after `cts.Cancel()`, if the STA thread is alive, use `staThread.Join(3000)` — if the ApplicationContext's onExit callback was wired to `cts.Cancel()`, then on Ctrl+C you need a way to tell the STA thread to exit.

    **Recommended approach:** Register a cancellation callback that invokes `Application.ExitThread()` on the STA thread:
    ```csharp
    // After creating the ApplicationContext but before Join:
    cts.Token.Register(() =>
    {
        // This may run on any thread — Application.ExitThread() is thread-safe
        Application.ExitThread();
    });
    ```

    Actually, place `cts.Token.Register(() => Application.ExitThread())` AFTER the STA thread starts but before the wait. This ensures that whether shutdown comes from Ctrl+C or tray Exit, the STA thread's message pump exits.

    Return 0 on clean exit.

```csharp
var daemonCommand = new Command("daemon", "Run as a persistent background daemon with CAPSLOCK overlay hook");
var backgroundOption = new Option<bool>("--background", "Detach from console and run in background with tray icon only");

        return 2;

    bool background = parseResult.GetValue(backgroundOption);
    bool verbose = parseResult.GetValue(daemonVerboseOption);
    return DaemonCommand.Run(background, verbose);
});

rootCommand.Subcommands.Add(daemonCommand);
```

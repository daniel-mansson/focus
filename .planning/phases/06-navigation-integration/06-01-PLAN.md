Create `focus/Windows/Daemon/Overlay/ForegroundMonitor.cs` — a sealed, disposable class that wraps `SetWinEventHook(EVENT_SYSTEM_FOREGROUND)`.

```csharp
using System.Runtime.InteropServices;
using System.Runtime.Versioning;
using global::Windows.Win32;
using global::Windows.Win32.Foundation;
using global::Windows.Win32.UI.Accessibility;

namespace Focus.Windows.Daemon.Overlay;

/// <summary>
/// Detects foreground window changes via SetWinEventHook(EVENT_SYSTEM_FOREGROUND).
/// Must be installed and uninstalled on the STA thread (the thread with the message pump).
/// Callback fires on the STA thread when WINEVENT_OUTOFCONTEXT is used.
/// </summary>
[SupportedOSPlatform("windows6.0.6000")]
internal sealed class ForegroundMonitor : IDisposable
```

Implementation requirements:
1. Define constants manually (CsWin32 may not emit them as named values):
   - `private const uint EVENT_SYSTEM_FOREGROUND = 0x0003;`
   - `private const uint WINEVENT_OUTOFCONTEXT = 0x0000;`
   - `private const uint WINEVENT_SKIPOWNPROCESS = 0x0002;`

2. Fields:
   - `private readonly Action<HWND> _onChanged;` — callback injected via constructor
   - `private WINEVENTPROC? _proc;` — delegate stored in field (NOT local variable — GC will collect it)
   - `private GCHandle _procHandle;` — pins the delegate so GC cannot relocate it while Win32 holds a pointer
   - `private HWINEVENTHOOK _hook;` — handle returned by SetWinEventHook
   - `private bool _disposed;`

3. Constructor: `public ForegroundMonitor(Action<HWND> onForegroundChanged)` — stores callback, nothing else.

4. `public void Install()`:
   - Assign `_proc = Callback;`
   - Pin with `_procHandle = GCHandle.Alloc(_proc);`
   - Call `_hook = PInvoke.SetWinEventHook(EVENT_SYSTEM_FOREGROUND, EVENT_SYSTEM_FOREGROUND, HMODULE.Null, _proc, 0, 0, WINEVENT_OUTOFCONTEXT | WINEVENT_SKIPOWNPROCESS);`
   - MUST be called on STA thread.

5. `public void Uninstall()`:
   - If `_hook != default`: call `PInvoke.UnhookWinEvent(_hook); _hook = default;`
   - If `_procHandle.IsAllocated`: call `_procHandle.Free();`
   - Set `_proc = null;`
   - MUST be called on same STA thread as Install().

6. `private void Callback(HWINEVENTHOOK hook, uint eventType, HWND hwnd, int idObject, int idChild, uint eventThread, uint eventTime)`:
   - Call `_onChanged(hwnd);` — use the hwnd parameter directly (Pitfall 4: do NOT call GetForegroundWindow inside callback).

7. `public void Dispose()`: if not disposed, set disposed = true, call Uninstall().

**CRITICAL constraints from research:**
- Delegate MUST be stored in an instance field AND pinned with GCHandle.Alloc — NOT a local variable.
- Use `HMODULE.Null` (not HINSTANCE) for the third parameter when using WINEVENT_OUTOFCONTEXT.
- The WINEVENTPROC callback signature must match CsWin32's generated delegate type exactly. Check the generated delegate in obj/ if there are compilation issues — CsWin32 may generate it as `Windows.Win32.UI.Accessibility.WINEVENTPROC`.

Create `focus/Windows/Daemon/Overlay/OverlayOrchestrator.cs` — the central coordinator that connects CapsLockMonitor (worker thread), ForegroundMonitor (STA thread), and OverlayManager (STA thread).

```csharp
using System.Runtime.InteropServices;
using System.Runtime.Versioning;
using System.Windows.Forms;
using Focus.Windows;
using global::Windows.Win32;
using global::Windows.Win32.Foundation;
using global::Windows.Win32.UI.WindowsAndMessaging;

namespace Focus.Windows.Daemon.Overlay;

[SupportedOSPlatform("windows6.0.6000")]
internal sealed class OverlayOrchestrator : IDisposable
```

Implementation requirements:

**Fields:**
- `private readonly OverlayManager _overlayManager;`
- `private readonly FocusConfig _config;`
- `private readonly Control _staDispatcher;` — WinForms control used for cross-thread Invoke
- `private readonly ForegroundMonitor _foregroundMonitor;`
- `private readonly System.Windows.Forms.Timer _delayTimer;` — for optional activation delay (OverlayDelayMs)
- `private readonly System.Windows.Forms.Timer _fadeTimer;` — for fade-in/fade-out animation (Plan 02 will add fade logic; create the timer here but don't start it yet)
- `private bool _capsLockHeld;`
- `private float _fadeProgress;` — 0.0=transparent, 1.0=full opacity (Plan 02 will use this)
- `private bool _fadingIn;` — true=fading in, false=fading out (Plan 02 will use this)
- `private bool _disposed;`
- `private volatile bool _shutdownRequested;` — checked before Invoke to avoid ObjectDisposedException during shutdown

**Solo-window dim color constant:**
- `private const uint SoloDimColor = 0x30AAAAAA;` — ~19% opacity neutral gray (Claude's discretion)

**Constructor** `public OverlayOrchestrator(OverlayManager overlayManager, FocusConfig config)`:
- Store parameters.
- Create `_staDispatcher = new Control();` and force handle creation: `_ = _staDispatcher.Handle;`
- Create `_foregroundMonitor = new ForegroundMonitor(OnForegroundChanged);` and call `_foregroundMonitor.Install();`
- Create `_delayTimer = new System.Windows.Forms.Timer();` with `_delayTimer.Tick += OnDelayTimerTick;`
- Create `_fadeTimer = new System.Windows.Forms.Timer();` with `Interval = 16;` (Plan 02 will wire Tick handler)
- MUST be constructed on STA thread.

**Public methods (called from CapsLockMonitor worker thread):**

`public void OnCapsLockHeld()`:
- If `_shutdownRequested` return immediately.
- Call `_staDispatcher.Invoke(OnHeldSta);` wrapped in try/catch for ObjectDisposedException and InvalidOperationException (shutdown safety — see Research open question 1).

`public void OnCapsLockReleased()`:
- If `_shutdownRequested` return immediately.
- Call `_staDispatcher.Invoke(OnReleasedSta);` wrapped in same try/catch.

`public void RequestShutdown()`:
- Set `_shutdownRequested = true;`

**Private STA-thread methods:**

`private void OnHeldSta()`:
- Set `_capsLockHeld = true;`
- If `_config.OverlayDelayMs > 0`: start `_delayTimer` with `Interval = _config.OverlayDelayMs;` and `_delayTimer.Start();` — delay fires once, then stops.
- Else: call `ShowOverlaysForCurrentForeground();` directly.

`private void OnReleasedSta()`:
- Set `_capsLockHeld = false;`
- Stop `_delayTimer` if running (user released before delay elapsed — prevents spurious trigger).
- Call `_overlayManager.HideAll();`

`private void OnDelayTimerTick(object? sender, EventArgs e)`:
- Stop `_delayTimer;` (fire once only).
- If `_capsLockHeld`: call `ShowOverlaysForCurrentForeground();`

`private void OnForegroundChanged(HWND hwnd)`:
- If `!_capsLockHeld` return.
- Call `ShowOverlaysForCurrentForeground();` — instant reposition per user decision (no cross-fade).

**Core scoring method:**

`private void ShowOverlaysForCurrentForeground()`:
1. Enumerate windows: `var enumerator = new WindowEnumerator();`
2. `var (windows, _) = enumerator.GetNavigableWindows();`
3. Apply exclude filter: `var filtered = ExcludeFilter.Apply(windows, _config.Exclude);`
4. Track candidates found: `int candidatesFound = 0;`
5. For each direction in `{ Direction.Left, Direction.Right, Direction.Up, Direction.Down }`:
   a. `var ranked = NavigationService.GetRankedCandidates(filtered, direction, _config.Strategy);`
   b. If `ranked.Count == 0`: call `_overlayManager.HideOverlay(direction);` and continue. (OVERLAY-05)
   c. Increment `candidatesFound`.
   d. Get top candidate: `var top = ranked[0].Window;`
   e. Build RECT from WindowInfo bounds: `var bounds = new RECT { left = top.Left, top = top.Top, right = top.Right, bottom = top.Bottom };`
   f. Call `_overlayManager.ShowOverlay(direction, bounds);` (uses configured direction color)
6. **Solo-window indicator** (OVERLAY-05 special case): if `candidatesFound == 0`:
   a. Get foreground window HWND: `var fgHwnd = PInvoke.GetForegroundWindow();`
   b. Get its bounds via `PInvoke.DwmGetWindowAttribute(fgHwnd, DWMWINDOWATTRIBUTE.DWMWA_EXTENDED_FRAME_BOUNDS, ...)` — use same pattern as existing code in Program.cs debug overlay command.
   c. If bounds retrieval succeeds, show all four overlays on the foreground window with `SoloDimColor`: `_overlayManager.ShowOverlay(direction, fgBounds, SoloDimColor);` for each direction.
   d. If bounds retrieval fails, do nothing (graceful degradation).

**Note on WindowInfo bounds:** WindowInfo has `Left`, `Top`, `Right`, `Bottom` int properties representing DWMWA_EXTENDED_FRAME_BOUNDS.

**Dispose:**
- Dispose `_foregroundMonitor` (must be on STA thread — and it will be, since OverlayOrchestrator.Dispose is called from STA shutdown path).
- Dispose/stop `_delayTimer` and `_fadeTimer`.
- Dispose `_staDispatcher`.

**CRITICAL anti-patterns to avoid:**
- Do NOT call OverlayManager methods from the CapsLockMonitor worker thread. Always marshal via `_staDispatcher.Invoke`.
- Do NOT call GetForegroundWindow inside the WinEvent callback (Pitfall 4) — but note that ShowOverlaysForCurrentForeground uses GetForegroundWindow indirectly via NavigationService (which gets the foreground window to exclude it from candidates). This is fine because by the time Invoke executes on STA, the foreground state is settled.
- Wrap `_staDispatcher.Invoke` in try/catch for shutdown races.
